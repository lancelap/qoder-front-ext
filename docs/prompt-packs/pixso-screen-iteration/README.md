# Пакет промптов: поэтапная реализация экрана по Pixso

Этот пакет нужен для задач, где:

- есть Pixso-макет или экспорт;
- не хочется запускать полный `/work:orchestrate`;
- нужно вести Qwen короткими итерациями с минимальным scope;
- важно, чтобы каждая итерация была легко проверяема и не раздувала diff.

Рекомендуемый порядок:

1. `00-validate-spec.md`
2. `01-plan.md`
3. `02-layout.md`
4. `03-filters.md`
5. `04-tabs-table.md`
6. `05-selection-states.md`
7. `06-polish-review.md`

Если после реализации появились точечные расхождения, используйте correction pass:

8. `07-fix-mapping.md`
9. `08-fix-behavior.md`
10. `09-fix-visual-alignment.md`

Как использовать:

1. Откройте нужный markdown-файл.
2. Подставьте свои пути к спеке, DSL, Pixso-ссылке и файлам проекта.
3. Вставьте текст в Qwen.
4. После каждого этапа визуально и технически проверяйте результат, только потом переходите к следующему.

Правило:

- один промпт = один изолированный этап;
- не просите в одном сообщении и layout, и таблицу, и wiring, и review;
- если этап дал лишние изменения, откатывайте scope в следующем промпте, а не расширяйте задачу.
- если реализация уже почти готова, не запускайте всё заново: используйте correction prompt на конкретный mismatch.
